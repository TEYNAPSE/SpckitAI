2. 서비스 형태 구체화 
    - 단순히 gpt나 재미나이 질문을 통해 정보를 얻었을 때 표 형식 정도로 구체화 되어 나옴
    - 정보의 정확성, 신뢰성은 있음 → 실시간 검색을 하는 쿼리가 내장되어있기 때문
    - 기존의 방식에서 문제
        - 시각적인 정보 제공이 부족하다고 느낌: 비 전문가의 경우 조립을 어떻게 하는지에 대한 가이드가 부족하여 유튜브 가이드나 전문가 블로그를 다시 참조 해야한다.
        - 전문 견적 사이트의 경우 ui/ux 에서부터 진입장벽이 생김
        
        (아래는 기존 견적 사이트들의 웹 화면)

    - 서비스 제공 전략:
        - 질문-대화를 통해 단계별로 제품 결정
        - 시각적 요소를 적극적 제공: 고른 제품이 어떤 위치에, 어떻게 생겼는지 점차 완성되면 사용자에게 높은 이해 제공 가능, + 재미있는 경험 제공 가능
            - 제품을 고를 때 마다 획일하게 모델링된 뼈대(스켈레톤) 점차적으로 완성됨.
            - 부품의 위치에 선과 제품 이미지를 띄워서 상세 정보와 설명을 알 수 있도록 함. (아래는 완성된 모습 예시)
            
            ![image.png](image%204.png)
            
        - 구현 가능성 검토:
            - 3D 에셋 제작: 각 주요 부품 별 통일된 하나의 에셋만 제작하면 되고, 제작역시 AI 툴을 이용해 쉽게 가능
            - 3D 프론트엔드 시각화: Babylon.js, three.js 와 같은 3d 시각화 라이브러리 이용
            - 백엔드: 호완성, 간섭등의 로직 구현 난이도
